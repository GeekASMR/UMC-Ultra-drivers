🐛 ioctl_spy5 崩溃排查 — NULL 回调指针
spy5 在运行 0.2 秒后进程闪退。经过 CDB 调试器分析，发现根因：
我们在 asioMessage(kAsioSupportsTimeInfo) 中响应了支持 TimeInfo， 但 ASIOCallbacks 中的 bufferSwitchTimeInfo 指针为 NULL。
官方驱动的内部线程解引用空指针 → 0xC0000005 访问违规 → 进程崩溃。
修复：正确提供 bufferSwitchTimeInfo 回调函数。

#Bug根因修复方案
1 输出 DMA 被内核清零 缺少 WAIT_FOR_BUFFER ACK 回应 在回调后发送 WFB IOCTL 作为确认
2 输入 ADC 采样异常/杂音 WFB 干扰 ADC 采样时序 先 Event 等待保护 ADC，再 WFB ACK
3 CPU 100% 死锁空转 轮询线程无阻塞等待 WaitForSingleObject(Event) 挂起线程
4 OVERLAPPED 竞态条件 主线程和轮询线程共用 OVERLAPPED 轮询线程使用独立 Event + OVERLAPPED
5 Studio One 性能表 100% ASIO ABI hi:lo 格式不兼容 getSamplePosition 按 hi/lo 分写
6 WFB OVERLAPPED 内存损坏 50ms 超时导致 OVERLAPPED 结构被复用 WFB 非阻塞 + CancelIoEx
7 CancelIo 无法跨线程取消 CancelIo 只能取消当前线程的 IO 升级为 CancelIoEx
